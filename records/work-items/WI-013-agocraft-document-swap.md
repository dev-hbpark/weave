## Summary

agocraft WI-012 (Phase A~C) 가 완성되어 `@agocraft/editor` 가 state pattern + DocumentType registry + React adapter 를 제공한다. 현재 weave/apps/web 은 자체 `apps/web/src/document/types.ts` 의 local Document model 을 useDocument 훅으로 관리하고, agocraft 와 연결은 `EditorProbe` 의 별도 mount 로만 존재 (Phase C). 이 WI 는 그 격차를 닫는다 — weave 의 local Document 를 `@agocraft/core` Document 로 마이그레이션하여 BehaviorEditor / Renderer / PresentPage 가 모두 agocraft surface 경유로 동작하게 한다.

## Why now

- agocraft Editor 가 production-ready surface (`createEditor`, `EditorProvider`, state machine, History, Plugin lifecycle) 를 제공 — 더 이상 mock state 로 우회할 이유 없음.
- weave 의 local Document 와 agocraft Document 는 model gap (flat items vs tree, behaviors vs units, schemaVersion 별 storage) 이 누적되고 있음 — 늦을수록 swap 비용 증가.
- WI-011 (Selection / Manipulation framework) 의 agocraft 측 `@agocraft/manipulation` graduate 가 이 swap 이후라야 자연스럽게 weave 측에서 소비 가능.

## Model gap

| 측면 | weave local | `@agocraft/core` |
|---|---|---|
| Document shape | `{ id, title, items[], updatedAt, schemaVersion: 3 }` | `{ id, schema, root: Item, meta }` |
| Item shape | `{ id, kind, attrs, behaviors[], createdAt }` | `{ id, kind, attrs, units[], children[], meta }` |
| Composition | flat array on Document | tree via `root.children` |
| Side-tables | `behaviors` (camera-target / hotspot / reveal-on-step) | `units` (typed payloads) |
| Time | `createdAt` per item | `meta.{createdAt, updatedAt, schemaVersion}` |
| Persistence | localStorage 직접 | ChangeStream + sink |

### Mapping decisions (DR-001-weave 후속 발행)

- weave 의 flat `items` → `root.children` (root.kind = "weave-doc")
- weave 의 `behaviors[]` → 별도 Unit kinds (CameraTargetUnit / HotspotUnit / RevealOnStepUnit) 또는 sidecar registry. Phase 1 에선 Item 의 `units` 배열에 옮겨 담음.
- weave 의 `CanvasShape` (shapes 배열) → canvas Item 의 children (각 shape 가 sub-Item) 또는 그대로 attrs.shapes 유지. Phase 1 에선 후자 (attrs 안에 array 보존).
- schemaVersion 별 storage → `@agocraft/core` 의 schema 도구 활용.

## Phased plan

### Phase 1 — Adapter / 양방향 mirror (이 라운드)

Goal: 기존 useDocument 의 API surface (`addItem`, `removeItem`, `updateBehavior`, `updateItem`, `updateShape`, `removeShape`, `reset`) 은 그대로 유지하면서, 내부에서 `@agocraft/core` Document 를 mirror 생성. EditorProvider 가 그 Document 를 host. 모든 e2e PASS.

Acceptance:
- [ ] `apps/web/src/document/agocraft-mirror.ts` — weave Document ↔ agocraft Document mapping functions + tests
- [ ] `useDocument` 가 내부에서 agocraft Document mirror 를 유지하고 외부에서 가져갈 수 있게 노출 (`docInAgocraft`)
- [ ] `DemoDocPage` 가 page 최상단에서 `EditorProvider` mount, mirror doc 을 host. EditorProbe 는 제거 또는 그대로 (compat 검증용).
- [ ] 14 e2e 모두 PASS
- [ ] typecheck + vite build PASS

### Phase 2 — Renderer / BehaviorEditor → agocraft surface 소비

Goal: weave 의 DOMAIN_RENDERERS 와 BehaviorEditor 가 weave local Item 대신 agocraft Item 을 직접 받아서 렌더. useDocument 의 API 는 `editor.exec` 으로 backed (commands).

Acceptance:
- [ ] 도메인별 renderer 의 props shape 변경 (`item: Item<agocraft>`)
- [ ] BehaviorEditor 가 agocraft Item 의 units (또는 sidecar) 에서 behaviors 읽기
- [ ] addItem / removeItem 등이 `editor.exec("item.add", …)` 로 routed
- [ ] localStorage persistence 가 `editor.changeStream` sink 로 routed

### Phase 3 — Local Document type 제거 + Manipulation graduate 연동

Goal: `apps/web/src/document/types.ts` 의 local Document 폐기. `@agocraft/manipulation` 가 graduate 되면 SelectionLayer 의 manipulation 도 agocraft surface 로.

Acceptance:
- [ ] `apps/web/src/document/types.ts` 제거 또는 type 만 남기고 implementation 제거
- [ ] WI-011 의 manipulation 이 `@agocraft/editor` 의 manipulationCapabilities path 로 routed
- [ ] schemaVersion bump 또는 storage migration 박제
